```python
def append_episode(self, record: EpisodeRecord) -> None:
    path = self.traces_dir / "episodes.jsonl"
    with open(path, "a", encoding="utf-8") as f:
        f.write(json.dumps(record.to_dict()) + "\n")

def load_episodes(self) -> list[EpisodeRecord]:
    path = self.traces_dir / "episodes.jsonl"
    if not path.exists():
        return []
    records = []
    for line in path.read_text(encoding="utf-8").splitlines():
        if line.strip():
            records.append(EpisodeRecord.from_dict(json.loads(line)))
    return records
```

## Task 2: Add TaskQueueEntry dataclass
**Action**: Add `TaskQueueEntry` dataclass to `homunculus/models.py` for queue item tracking.

The dataclass should wrap `GeneratedTask` with queue-specific metadata:
- `task_id: str` — Unique identifier (matches GeneratedTask.task_id)
- `task: GeneratedTask` — The actual task
- `queued_at: str` — UTC timestamp when added to queue
- `status: str` — "pending" | "in_progress" | "completed" | "failed"
- `attempts: int` — Number of execution attempts (default 0)
- `last_error: str | None` — Error message from last attempt (None if no errors)
- `completed_at: str | None` — UTC timestamp when completed (None if pending)
- `outcome: str | None` — "accepted" | "reverted" | "error" (None if pending)

Include `to_dict()` and `from_dict()` methods following existing patterns.

## Task 3: Add queue persistence to storage.py
**Action**: Add these methods to `ArtifactStore` class in `homunculus/storage.py`:

1. `append_to_queue(self, entry: TaskQueueEntry) -> None`
   - Appends to `runtime/task_queue.jsonl`
   - Follow `append_episode` pattern

2. `load_queue(self) -> list[TaskQueueEntry]`
   - Loads from `runtime/task_queue.jsonl`
   - Returns only entries with status="pending"
   - Follow `load_episodes` pattern

3. `update_queue_entry(self, task_id: str, status: str, outcome: str | None = None) -> None`
   - Reads entire queue, updates matching entry, rewrites file
   - Sets `completed_at` to `utc_now()` if status is "completed" or "failed"
   - Atomic write using temp file + os.replace pattern

4. `archive_completed_tasks(self) -> int`
   - Moves completed/failed entries from task_queue.jsonl to task_history.jsonl
   - Returns count of archived entries
   - Keeps task_queue.jsonl lean (only pending/in_progress)
