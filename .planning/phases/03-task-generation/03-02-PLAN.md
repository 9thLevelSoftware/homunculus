## Mode-Specific Findings Formats

### MetricsMode findings
```python
{"type": "rate", "name": "success_rate", "value": 0.65, "threshold": 0.7, "status": "warning"}
{"type": "rate", "name": "error_rate", "value": 0.15, "threshold": 0.1, "status": "critical"}
{"type": "distribution", "name": "failure_stage", "stage": "execute", "count": 5}
```

### CritiqueMode findings
```python
{"type": "pattern", "area": "error_handling", "description": "...", "impact": "high"}
{"type": "weakness", "area": "async_code", "description": "...", "recommendation": "..."}
{"type": "strength", "area": "testing", "description": "..."}
```

### CoverageMode findings
```python
{"type": "coverage", "file": "module.py", "line_rate": 0.45, "status": "low"}
{"type": "todo", "file": "module.py", "line": 42, "text": "TODO: implement X"}
{"type": "gap", "file": "module.py", "description": "No tests for error paths"}
```

### ComparativeMode findings
```python
{"type": "pattern", "observation": "Winning patches are smaller", "confidence": 0.8}
{"type": "comparison", "winner_avg_lines": 15, "loser_avg_lines": 45}
{"type": "insight", "area": "planning", "description": "More plan steps correlate with failure"}
```

## Task 2: Implement mode-specific generators with defensive parsing
**Action**: Add private methods for each introspection mode with defensive dict access.

**CRITICAL**: Each mode produces different finding structures. Use `.get()` with defaults:

```python
def _extract_finding_field(self, finding: dict, field: str, default: Any = None) -> Any:
    """Safely extract field from finding dict."""
    return finding.get(field, default)

def _infer_severity(self, finding: dict) -> str:
    """Infer severity from finding status or type."""
    # Check explicit status field first
    if status := finding.get("status"):
        return status  # "critical", "warning", "info"
    # Infer from finding type
    if finding.get("type") == "weakness":
        return "high"
    if finding.get("type") == "gap":
        return "medium"
    return "low"
```

1. `_generate_from_metrics(self, result: IntrospectionResult) -> list[GeneratedTask]`
   - Check for: `finding.get("type")`, `finding.get("status")`, `finding.get("value")`
   - Critical rates (status="critical") → high priority practice tasks
   - Warning rates (status="warning") → medium priority improvement tasks
   - Example: error_rate > 0.1 → "Practice error handling patterns"

2. `_generate_from_critique(self, result: IntrospectionResult) -> list[GeneratedTask]`
   - Check for: `finding.get("type")`, `finding.get("area")`, `finding.get("description")`
   - Weaknesses → targeted practice tasks
   - Patterns → habit-forming tasks
   - Example: weakness in async_code → "Implement async file operations"

3. `_generate_from_coverage(self, result: IntrospectionResult) -> list[GeneratedTask]`
   - Check for: `finding.get("type")`, `finding.get("file")`, `finding.get("line_rate")`
   - Low coverage files → "Add tests for {file}"
   - TODOs → "Complete TODO in {file}:{line}"
   - Gaps → "Add error path tests for {file}"

4. `_generate_from_comparative(self, result: IntrospectionResult) -> list[GeneratedTask]`
   - Check for: `finding.get("type")`, `finding.get("observation")`, `finding.get("confidence")`
   - Patterns → "Practice {pattern} approach"
   - Insights → habit-forming tasks

## Task 3: Implement task prompt templates
**Action**: Create clear, actionable prompts for each task type.

Use these prompt templates:

**Metrics-based**:
```
# Practice: {area}

Your {metric_name} is at {value:.0%}, below the healthy threshold of {threshold:.0%}.

## What
Write code that exercises {area} patterns. Focus on:
- {specific_pattern_1}
- {specific_pattern_2}

## Success Criteria
The next introspection cycle should show improvement in {metric_name}.
```

**Critique-based**:
```
# Improve: {area}

A pattern was identified: {description}

## What
{recommendation}

## Success Criteria
- Tests pass
- The identified pattern should not recur in future episodes
```

**Coverage-based**:
```
# Add Tests: {file}

Current line coverage: {line_rate:.0%}

## What
Add tests covering:
{gap_description}

## Success Criteria
- Line coverage for {file} increases to at least {target_rate:.0%}
- All new tests pass
```

**Comparative-based**:
```
# Apply Lesson: {insight_area}

Analysis of winning vs losing patches shows: {observation}

## What
Apply this insight to improve patch success rate.

## Success Criteria
- Future patches follow the identified pattern
```

## Task 4: Add priority calculation
**Action**: Implement `_calculate_priority(self, finding: dict, mode: str) -> float`

Priority formula based on finding type:
- Critical severity (error_rate > threshold): 0.9-1.0
- Warning severity: 0.6-0.8
- Informational: 0.3-0.5
- Suggestions: 0.1-0.3

Mode weights:
- metrics: 1.0 (most actionable)
- critique: 0.9 (LLM-identified issues)
- coverage: 0.7 (concrete but lower urgency)
- comparative: 0.6 (insights, not urgent)

Final priority = base_severity * mode_weight
