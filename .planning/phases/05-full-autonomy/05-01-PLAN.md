<task id="05-01-3">
<title>Implement watchdog.py + daemon integration</title>
<action>
Per spec §5:

1. `homunculus/autonomy/watchdog.py`:
   - `Watchdog` class constructed with `state_path: Path` (default `runtime/watchdog.json`).
   - Methods: `tick(cycle_outcome: CycleOutcome) -> None`, `record_task_revert(task_id: str) -> None`, `merge_failures(count: int) -> None`, `active_flags() -> list[str]`, `load() -> WatchdogSnapshot`, `save() -> None`.
   - Atomic persistence: write to `watchdog.json.tmp`, `os.replace()` to `watchdog.json` (match `storage.py` pattern — grep `os.replace` to find reference).
   - Corrupt JSON recovery: on `json.JSONDecodeError` during load, log a warning and start with a fresh `WatchdogSnapshot`; never crash the daemon.
   - Flag derivation in `active_flags()`:
     - `cycle_failure:{N}` when `consecutive_cycle_failures >= FAILURE_THRESHOLD_CYCLE`
     - `merge_failure:{N}` when `consecutive_merge_failures >= FAILURE_THRESHOLD_MERGE`
     - `repeat_revert:{task_id}` for every entry in `repeated_task_reverts` with value `>= FAILURE_THRESHOLD_TASK_REVERT`
2. Integrate into `homunculus/daemon.py`:
   - Construct `Watchdog` in `Daemon.__init__` using `runtime_dir / "watchdog.json"`.
   - Integration point: grep `daemon.py` for the cycle-end location. Likely methods: `run_once()`, `_execute_cycle()`, `run_continuous()` inner loop. Pick the narrowest method that runs ONCE per cycle and has the outcome in scope. If no single cycle-end line exists, introduce one minimal helper `_finalize_cycle(outcome)` that existing code calls — purely additive, no behavior change.
   - Call `self._watchdog.tick(cycle_outcome)` where `cycle_outcome` is the existing outcome value OR a small adapter dict `{"status": "success|failed"}` if no dataclass exists.
   - Pull merge failure count from existing evolution counter via `storage.load_evolution_state` or equivalent. Do NOT mutate the evolution counter.
   - Preserve backward compatibility: all existing daemon tests must still pass. If `runtime_dir` doesn't exist in a test fixture, watchdog should no-op.

CRITICAL: do not change the signature of any existing daemon method. The watchdog is additive. Grep `tests/test_daemon.py` for constructor calls — if tests instantiate `Daemon(...)` without the runtime_dir present, make sure the watchdog constructor creates the parent dir or tolerates absence.
</action>
