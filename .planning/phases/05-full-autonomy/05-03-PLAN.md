<task id="05-03-1">
<title>Write SOAK-PROTOCOL.md and execute soak run</title>
<action>
1. Write `.planning/phases/05-full-autonomy/SOAK-PROTOCOL.md` with the full runbook:
   - Branch naming: `phase-5/soak-YYYYMMDD` created from `master`
   - **Throughput pre-check** (addresses CRITICAL-1 — SC3 data-starvation risk):
     - Read last 14 days of `traces/episodes.jsonl` to compute average episodes/day
     - Read `[training].min_samples_for_train` and `[evolution].min_loras_for_merge` from config
     - Compute projected LoRAs-merged over 7 days: `floor((episodes_per_day * 7 * success_rate) / min_samples_for_train) / min_loras_for_merge`
     - If projection < 1: PAUSE soak, lower thresholds in config (commit BEFORE soak starts) OR extend soak duration until projection ≥ 1.5 (safety margin)
     - Record computed projection in `soak-log/day-00-baseline.json` for audit
   - Pre-run: `python -m homunculus.cli autonomy-preflight --config homunculus.toml` — MUST exit 0
   - Baseline capture: `python -m homunculus.cli autonomy-report --config homunculus.toml --json > .planning/phases/05-full-autonomy/soak-log/day-00-baseline.json`
   - Start command: `python -m homunculus.daemon --config homunculus.toml` (foreground; can use `nohup` or process supervisor)
   - Daily observation cadence:
     - Run `autonomy-report --json > soak-log/day-NN.json` each day
     - Write a markdown summary `soak-log/day-NN.md` with notable events (new task generated, episode reverted, LoRA trained, merge executed, watchdog flag raised)
     - Diff against previous day's JSON for quick visual scan
   - Abort conditions (human decision, no code changes during run):
     - Watchdog flag `cycle_failure:3+` persistent ≥ 2 consecutive cycles
     - Disk usage > 90% on artifact volume
     - Test suite regression detected between any two days
     - Git corruption (e.g., `git fsck` failure)
   - Hands-off rule: ZERO code edits, ZERO config edits, ZERO manual commits during the soak window. Only the daily report commands run.
   - End-of-run: ≥7 full days elapsed AND baseline+final reports captured.
2. Execute the soak run per the protocol.

   **Execution model** (addresses CRITICAL-2 — wall-clock handoff):
   - This task is NOT a single build-agent session. The agent starts the soak then yields.
   - Start daemon under a process supervisor (`nohup`, `pm2`, `systemd --user`, or Windows `Start-Process`) so it survives terminal close.
   - Record PID + start timestamp in `soak-log/day-00-process.json`.
   - Schedule daily observation via OS cron / Task Scheduler OR a lightweight Python script running in a second tmux/screen pane:
     - Daily command: `python -m homunculus.cli autonomy-report --config homunculus.toml --json > soak-log/day-$(date +%d).json`
     - Append markdown summary via diff-between-days
   - The executing legion agent RETURNS after scheduling. It does NOT block for 7 days.
   - Resume: after ≥7 days wall-clock, the user or a new legion session invokes 05-03-2.

3. Save daily artifacts under `.planning/phases/05-full-autonomy/soak-log/`.

If an abort condition trips: halt the soak (stop the daemon via its PID), capture state, generate an introspection task for the failure, and re-plan. Do NOT silently restart.
</action>

- At least 7 `day-NN.json` files (NN=01..07) exist, each representing a 24-hour separation (timestamps in JSON confirm)

<task id="05-03-2">
<title>Run autonomy-accept + produce 05-ACCEPTANCE.md</title>
<action>
1. After soak completes, run:
```
python -m homunculus.cli autonomy-accept \
    --config homunculus.toml \
    --soak-log .planning/phases/05-full-autonomy/soak-log \
    --soak-branch phase-5/soak-YYYYMMDD \
    --output .planning/phases/05-full-autonomy/05-ACCEPTANCE.md
```
2. Inspect the generated report. Verify each criterion:
   - SC1: uptime ≥ 168 hours (7 days * 24)
   - SC2: ≥10 self-directed tasks (source ∈ {generated, resonance}, outcome=success)
   - SC3: ≥1 LoRA trained AND ≥1 merged AND lineage generation incremented
   - SC4: unittest discover exit 0 at acceptance time
   - SC5: `patch_success_rate_trend >= -0.02` AND (`coverage_trend` is None or `>= -0.02`)
   - SC6: git log on soak branch shows only agent authorship
3. If overall == PASS: proceed to 05-03-3.
4. If overall == FAIL: do NOT edit the report. Capture the verdict, file a new phase/task via `python -m homunculus.cli` task-queue addition OR a suggestion under `suggestions/` describing the failed criterion, and HALT. A failure here means Phase 5 is not yet complete; re-plan (another soak, or specific mitigation).

Record the commit SHA range of the soak run at the top of the acceptance report for traceability.
</action>
